Length

Longer passwords are indicative to good password strength.

Aim for longer passwords when possible, as each additional character will increase your password strength exponentially.

A password length of 16-20 characters is considered very strong and have a low chance of being guessed.

Complexity

Use a mix of lowercase and uppercase letters, digits, and symbols when making passwords.

A varied character set improves password strength by increasing the number of different characters which could potentially be found in the password.

Uniqueness

Do NOT reuse passwords!

Unknown to you, password leaks might occur to databases storing your passwords.

Repeating old passwords puts your information at risk, as your passwords are now known to have been used previously.

Use unique passwords for each application or device.

Things to Bear in Mind

  • Try to use passphrases when making passwords, with four to five distinct words separated by white spaces if possible. It will make the password much harder to guess and a lot easier for you to remember.


  • Find a nice balance between long and complex passwords. Very secure passwords are ideal, but if you have to remake them every once in a while because they are hard to remember, you might get a more tedious and annoying relationship with passwords, which could impact password security in the future.

In today's society, online threats evolve as quickly as technology itself. Malicious parties are able to operate more sophisticated than ever before, as new tools are made and new exploits are discovered. Two essential tools that help securing your information and add more layers of protection are Multi-Factor Authentication (MFA) and Password Managers.

What Is Multi-Factor Authentication

Multi-factor authentication (MFA) adds an extra layer of protection to ensure that compromising a password is not enough to get authentication to user accounts. It is widely supported and is offered as a free security option.

Not all applications or devices offer MFA as a securiy option, but it is strongly recommended to utilize it when possible. After the user has entered the correct login credentials, the user will be required to provide more information to confirm their identity.

Methods of Confirming Your Identity

  • Authentication using biometrics
  • A prompt on a trusted advice
  • A code sent to a phone or app

What Is a Password Manager and Why Is It Used?

A password Manager is a tool that can help you generate strong and unique passwords while storing them in an encrypted database. It can provide the security benefits of large and complex passwords while not having to remember all your passwords individually. All you need to remember is the master password for logging into the password manager.

The strong passwords it creates helps users maintain a healthy password hygiene. However, it might lead to a single point of failure as all passwords are stored in a single database behind one set of credentials.

Brute-Force Attack:

Systematically guesses various password combinations.

These attacks are simple, requiring expending computational power to launch an attack, but can take a lot of time and cannot be done offline.

Dictionary Attack:

A type of brute-force attack which uses a preprepared dictionary to base the guesses on.

Dictionary attacks threatens the use of unmodified trivial and simple passwords.

Botnets

A possibly vast network of infected computers.

These networks controlled by an attacker allow for a large amount of attempts at guessing passwords, reducing the estimated time it takes for a password to break.

List-Based Attack:

A type of brute-force attack using lists of previously leaked passwords.

Aims to gain an advantage by exploiting previous information about user password hygiene.

Consequences of Poor Password Hygiene ‚ùå

Most data breaches targeting individual user accounts occur due to phishing attacks or similar means by tricking victims into giving up sensitive information. This is due to these attacks being much easier to execute, and on a large enough scale there is a very high success rate. Data leaks from otherwise reputable industry actors is also a major reason why users' login credentials can be known to malicious parties, mainly due to these large databases hosting a vast amount of user data. This exposes a large number of credentials if these types of databases are ever breached.

These are some of the possible consequences that weak passwords and poor password hygiene can have on the end user.

Monetary

Accounts controlled by attackers and connected to payment processers might issue fraudulent purchases, which can be very costly for the victims of such attacks.

Banks and other relevant financial bodies should be quickly informed in the event of a registered password breach in order to freeze the affected accounts.



Psychological

Identity theft is broader in definition, but generally concerns the implications of an attacker gaining unauthorized access to a victim's credentials and pretending to be an authentic user.

Losing control over user accounts and sensitive information, or being the victim of stalking or blackmailing might cause feelings of distress and helplessness in the victim.
